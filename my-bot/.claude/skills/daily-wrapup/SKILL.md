# Skill: Daily Wrap-Up

**Name:** daily-wrapup
**When to use:** Job title contains "daily wrap-up", "end of day summary", or "wrap up today". Also triggered automatically at 6 pm weekdays and when a file in notes/ is saved.

---

## Steps

1. **Find today's notes**
   - Look in `notes/` for any files modified today (check file dates)
   - Also check for any files named with today's date (e.g. `2026-07-08.md`, `notes-2026-07-08.txt`)
   - If no notes folder or no files today: write summary with "No notes found today."

2. **Read the notes**
   - Read each file found
   - Extract the key points: what was completed, what is in progress, what comes next

3. **Write the summary**
   - Create or append to `log/YYYY-MM-DD-wrapup.md` (use today's date)
   - Use this exact format:

```
# Daily Wrap-Up — YYYY-MM-DD

## Done
- [one line per completed item]

## Doing
- [one line per in-progress item]

## Next
- [one line per upcoming item]

---
*Generated by Barry at HH:MM*
```

   - If a section has nothing, write "- Nothing noted today."
   - Keep each bullet to one line — plain and clear

4. **Mark the job done** (do not mark "needs my OK" — this is a write-to-file task, not a message)

---

## Quality bar
- Date in filename and heading must match today's date exactly
- No invented items — only what is in the notes
- If notes/ folder does not exist: note it clearly in the summary, do not fail
- Always create the `log/` folder if it does not exist

---

## Example output

```
# Daily Wrap-Up — 2026-07-08

## Done
- Completed enrollment form review for 3 new students
- Sent fee reminder to partner organisations

## Doing
- Waiting for birth certificate copies from 5 families

## Next
- Follow up with Mrs. Fatuma re: Grade 3 enrollment
- Prepare welcome letters for new intake

---
*Generated by Barry at 18:02*
```
